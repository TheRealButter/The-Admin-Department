# Pilot Readiness Pack

This folder prepares The Admin Department systems for Gate 3: private pilot readiness.

Gate 3 means the system is not only working in a sheet, but can be used safely with a controlled pilot client.

## Gate 3 requirements

A system becomes private-pilot ready only when:

1. Google Sheets live test passed.
2. Private client workspace is prepared.
3. Real client data is kept out of the public repo.
4. Access rules are defined.
5. Handover test is completed.
6. Pilot scope is narrow.
7. Weekly report process is clear.
8. Stop/go decision is documented.

## Included files

- `private-pilot-workspace-checklist.md`
- `pilot-access-control-template.md`
- `handover-test-script.md`
- `handover-scorecard.md`
- `first-pilot-operating-plan.md`
- `pilot-closeout-report-template.md`
- `gate-3-readiness-checklist.md`

## Rule

Do not use real client data in the public GitHub repo.

Real pilot data must live only in the private client-approved workspace.
