# Department System Installation Checklist

Use this checklist for every paid setup.

## 1. Client setup

- [ ] Confirm business name
- [ ] Confirm primary contact
- [ ] Confirm department being installed
- [ ] Confirm package and price
- [ ] Confirm start date
- [ ] Confirm internal owner on client side

## 2. Workflow discovery

- [ ] Identify current intake sources
- [ ] Identify current tracker/spreadsheet if any
- [ ] Identify current folder/document structure
- [ ] Identify statuses currently used
- [ ] Identify who approves what
- [ ] Identify recurring delays/missed tasks
- [ ] Identify reporting needs

## 3. System configuration

- [ ] Copy correct tracker template
- [ ] Add client-specific fields
- [ ] Add statuses
- [ ] Add owners/assignees
- [ ] Add priority rules
- [ ] Add next-action rules
- [ ] Add approval rules
- [ ] Add escalation rules
- [ ] Create folder structure
- [ ] Add message templates
- [ ] Add weekly report template

## 4. Demo data test

- [ ] Add 3–5 realistic sample records
- [ ] Move records through statuses
- [ ] Test missing-info scenario
- [ ] Test overdue next-action scenario
- [ ] Test approval scenario
- [ ] Test weekly report output

## 5. Client handover

- [ ] Walk client through tracker
- [ ] Explain statuses
- [ ] Explain daily routine
- [ ] Explain approval rules
- [ ] Explain escalation rules
- [ ] Explain weekly report
- [ ] Confirm who updates the system daily

## 6. Go-live checklist

- [ ] Remove/label demo data
- [ ] Add real first records
- [ ] Confirm folder links
- [ ] Confirm template wording
- [ ] Confirm report schedule
- [ ] Confirm support channel
- [ ] Confirm first review date

## 7. Retainer checklist

Weekly/monthly support may include:

- [ ] Review stuck records
- [ ] Clean statuses
- [ ] Improve templates
- [ ] Add missing fields
- [ ] Update report
- [ ] Fix workflow problems
- [ ] Add light automation only after manual process works

## Rule

If the client cannot run the workflow manually, do not automate it yet.
