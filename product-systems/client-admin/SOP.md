# Client Admin SOP

## Daily routine

### 1. Check new clients

A new client record must be created when:

- a proposal is accepted
- a deposit/payment is expected
- a service setup is approved
- a signed agreement is pending
- a new project or retainer starts

### 2. Send welcome message

Every new client should receive a welcome message with:

- what happens next
- what information/documents are needed
- who their contact person is
- expected next step/date

### 3. Create client folder

Recommended structure:

```text
Client Name /
  01 - Agreements /
  02 - Invoices and Payments /
  03 - Client Documents /
  04 - Delivery /
  05 - Reports /
```

### 4. Track required items

Typical checklist:

- signed agreement
- business details
- contact details
- billing details
- proof of payment/deposit
- required documents
- access details where applicable
- kickoff call scheduled
- internal handover complete

### 5. Follow up missing items

If a required item is missing, add:

- missing item
- person responsible
- follow-up date
- message sent
- next action

### 6. Status rules

Use one of:

- New Client
- Welcome Sent
- Documents Requested
- Waiting for Client
- Internal Setup
- Payment/Agreement Pending
- Ready to Start
- Active
- Stuck
- Cancelled

### 7. Mark onboarding complete

Only mark Active when:

- required documents are received
- folder exists
- agreement/payment status is confirmed if needed
- internal handover is complete
- next service/delivery date is clear

## Escalation rules

Escalate if:

- client is stuck for more than 5 business days
- payment/agreement is disputed
- client sends sensitive or unclear documents
- required access details are missing
- kickoff cannot proceed

## Weekly report

Report:

- new clients added
- clients activated
- stuck onboarding cases
- missing documents
- agreements/payment pending
- next actions by owner

## Quality checklist

- every new client has a status
- every stuck client has a reason
- every missing item has a follow-up date
- every active client has a folder link
- every onboarding record has an owner
