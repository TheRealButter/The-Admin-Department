# Client Admin Demo

This demo package is the third sellable product proof for The Admin Department.

It shows how messy client onboarding becomes a clear client admin workflow:

1. new clients are captured
2. welcome status is tracked
3. documents and missing items are visible
4. agreements and payment status are tracked
5. folder setup and internal handover are managed
6. weekly onboarding reporting is produced

## Demo files

- `demo-dashboard.html` — browser demo of the Client Admin system.
- `demo-dashboard.css` — styling for the demo.
- `onboarding-pipeline.csv` — sample client onboarding pipeline.
- `document-checklist.csv` — sample required document checklist.
- `filled-weekly-report.md` — realistic filled report for a demo client.
- `demo-walkthrough.md` — 5-minute sales demo script.
- `pilot-setup-checklist.md` — first paid pilot checklist.

## Demo positioning

Do not pitch this as a project management app or AI assistant.

Pitch it as:

> We set up a simple client admin system so onboarding, documents, payments, folders, and handovers stop depending on memory.

Automation can be added later, but the demo should first prove the workflow.
